## О проекте

Сейчас на проекте есть курсы, статьи и упражнения. Пользователи могут выполнять C# код прямо из браузера, код посылается на бек и там выполняется.
Упражнения работают аналогично, юзер пишет код, отправляет на проверку, прогоняются тесты, возвращается ответ.
Также уже реализована регистрация и аутентификация, адаптивный дизайн, тесты.
При разработке ориентируюсь на [leetcode](https://leetcode.com/problems/add-two-integers/description/).

## Особенности фронта

Контент в формате md храниться на беке, потом рендерится в html, компонент Md, используется https://github.com/unifiedjs/unified плюс плагины.
Для генерации апи используется swagger-typescript-api.
За логику взаимодействия с беком отвечает apiClient. Там достаточно сложная логика, есть тесты.
В качестве редактора кода используется Monaco editor, выполнения происходит на беке, запускается отдельный Docker контейнер, intellisence и ошибки тоже приходят с бека.
e2e тесты.

Для рендеринга markdown используется [react-markdown](https://github.com/remarkjs/react-markdown): markdown -> промежуточные шаги ... -> HTML
Подробнее [тут](https://github.com/remarkjs/react-markdown?tab=readme-ov-file#architecture)
Чтобы задать кастомный рендеринг, например для тега `<em>` в markdown, надо описать его рендер в components Markdown

## Как запустить бек

- установить postgres 15
- установить .net8
- установить Docker
- запустить Docker
- сбилдить образ исполнителя кода командой `make docker_build`
- сбилдить образ тестера упражнений командой `docker_tester_build`
- запустить бек кодандой `make run_back`
- сваггер будет доступен по ссылке http://localhost:5002/swagger/index.html
